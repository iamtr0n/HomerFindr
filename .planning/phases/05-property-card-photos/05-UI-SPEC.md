## Photo Area: State Machine

The photo area has exactly 3 states. State is tracked with a single `useState` boolean: `imgLoaded`.

**Decision (Claude's discretion, D-10):** Use `onLoad` to cancel the pulse. Placeholder renders with `animate-pulse` initially. When `onLoad` fires on the `<img>`, set `imgLoaded = true`, which removes `animate-pulse` from the placeholder. This gives users a clear "something loaded" signal while avoiding an eternal pulse on cards that successfully display photos.

### State 1 — Loading (image requested, not yet resolved)

The `<img>` element is present in the DOM but has not fired `onLoad`. The placeholder div is visible beneath it (hidden by `display: none` only after load; starts visible via sibling relationship).

**Implementation note:** The existing pattern keeps `<img>` and placeholder as adjacent siblings. The `onError` handler sets `img.style.display = 'none'` and `img.nextSibling.style.display = 'flex'`. To support the loading state, render placeholder visible by default when `photo_url` exists, and hide it via `onLoad`.

Placeholder div classes during loading:
```
w-full h-52 bg-slate-100 flex flex-col items-center justify-center text-slate-400 gap-2 animate-pulse
```

`<img>` classes during loading:
```
w-full h-52 object-cover bg-slate-100
```
`<img>` inline style during loading: `{}` (no override — visible)

### State 2 — Loaded (image loaded successfully)

`onLoad` fires. Set `imgLoaded = true`. Placeholder `animate-pulse` is removed. The `<img>` is visible. Placeholder remains in DOM but hidden (`display: none` set by ref or inline style via `imgLoaded` state).

Placeholder div classes after load:
```
w-full h-52 bg-slate-100 flex flex-col items-center justify-center text-slate-400 gap-2
```
Placeholder div inline style: `{ display: 'none' }`

`<img>` classes:
```
w-full h-52 object-cover bg-slate-100
```

### State 3 — Error / No Photo

Two entry paths:
1. `photo_url` is null/undefined — `<img>` is never rendered; placeholder renders without pulse.
2. `onError` fires on `<img>` — handler sets `img.style.display = 'none'` and `img.nextSibling.style.display = 'flex'`. Also set `imgLoaded = false` to restore placeholder visibility.

Placeholder div classes (no pulse in error state — image is not coming):
```
w-full h-52 bg-slate-100 flex flex-col items-center justify-center text-slate-400 gap-2
```
Placeholder div inline style: `{}` (visible by default when no `photo_url`; revealed by `onError` handler otherwise)

## Image Element Spec

Source: D-01, D-09, D-10.

Full attribute set for the `<img>` element when `photo_url` is present:

```jsx
<img
  src={photo_url}
  alt={address}
  className="w-full h-52 object-cover bg-slate-100"
  referrerPolicy="no-referrer"
  onLoad={() => setImgLoaded(true)}
  onError={(e) => {
    e.target.style.display = 'none'
    e.target.nextSibling.style.display = 'flex'
    setImgLoaded(true)
  }}
/>
```

- `referrerPolicy="no-referrer"` — suppresses `Referer` header to bypass CDN hotlink blocking on rdcpix.com (Realtor.com CDN). This is the fix for the 403s (D-01).
- `alt={address}` — unchanged from existing component.
- `className` — preserves existing `h-52 w-full object-cover`; `bg-slate-100` shows during image decode before pixels paint.
- `onLoad` — fires when image pixels are available; sets `imgLoaded = true` to remove pulse from placeholder.
- `onError` — unchanged behavior from existing component (D-09): hide `<img>`, reveal sibling placeholder div.

## State Management Approach

**Mechanism:** Single `useState` in `PropertyCard`.

```jsx
const [imgLoaded, setImgLoaded] = useState(false)
```

**Why not CSS-only:** The `animate-pulse` cancellation on load requires React state. CSS-only (e.g., `:has(img[loaded])` pseudo-selectors) is not reliably supported across all browsers and would require a data attribute toggle anyway — same cost, less readable.

**`imgLoaded` usage:**

| State | `imgLoaded` | Placeholder `display` | Placeholder has `animate-pulse` |
|-------|-------------|----------------------|--------------------------------|
| No `photo_url` | — (not used) | flex (default) | no |
| `photo_url` present, loading | false | flex (default) | yes |
| `photo_url` loaded | true | none (inline style) | no |
| `onError` fired | (reset to false) | flex (restored by onError handler) | no |

**onLoad cancels pulse:** Placeholder class string is computed:
```jsx
const placeholderClass = `w-full h-52 bg-slate-100 flex flex-col items-center justify-center text-slate-400 gap-2${photo_url && !imgLoaded ? ' animate-pulse' : ''}`
```

**onError handler:** Existing DOM-manipulation approach preserved (D-09). Handler does not need to call `setImgLoaded` — once `onError` fires, the img is hidden and the placeholder is shown via inline style on the nextSibling. The `animate-pulse` is already off because the placeholder div's class string is conditional on `photo_url && !imgLoaded`; once `onError` fires, `imgLoaded` is still false but the placeholder is revealed by the handler — the class string will not have `animate-pulse` if the developer also calls `setImgLoaded(true)` in the handler, OR the handler can explicitly strip the class. Simpler approach: call `setImgLoaded(true)` inside `onError` as well, so `animate-pulse` is removed whether the image loaded or errored.

**Updated `onError`:**
```jsx
onError={(e) => {
  e.target.style.display = 'none'
  e.target.nextSibling.style.display = 'flex'
  setImgLoaded(true)
}}
```

This ensures `animate-pulse` is gone in both success and error final states.
